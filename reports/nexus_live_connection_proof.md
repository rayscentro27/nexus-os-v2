## Exact Blockers

### Blocker 1: No Active UI Component Queries Supabase
- `src/services/db.ts` exists with full query capability
- No active component imports it
- A parallel Supabase-backed UI exists in `sections.tsx` but is not rendered

### Blocker 2: Ray Review Approvals Not Persisted
- `RayReviewCenter.jsx` saves decisions to localStorage key `nexus-ray-review-decisions-v2`
- Cards come from static `rayReviewData.js` (generated by Python from disk JSON)
- No Supabase write on approve/hold/reject
- Approvals disappear when localStorage is cleared

### Blocker 3: Hermes Has No Backend Context Endpoint
- `hermesSupabaseContextAdapter.ts` is a stub returning `false`
- No Netlify function or backend route exists for Hermes to query
- Edge function (`hermes-chat`) is a chat proxy, not a data query endpoint

### Blocker 4: Research Engine Does Not Write to Supabase from Frontend
- Python scripts write local JSON files
- Supabase inserts in `common.py` are gated on `sb.configured()` (server-side only)
- Frontend research panel reads static `researchEngineData.js`
