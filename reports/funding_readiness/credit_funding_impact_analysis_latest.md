# Credit Funding-Impact Analysis

Deterministic TypeScript and Python engines classify parsed information into high funding impact, utilization impact, report-item review, inquiry review, personal-information review, evidence needed, specialist exception, and no action suggested. A parsed flag is never described as verified inaccuracy.

The live fake report produced 31 funding-impact items, 3 inquiry reviews, 31 evidence-needed items, 28 specialist exceptions, and 0 no-action items. The exception count reflects incomplete bureau/furnisher attribution in many fixture fragments. It is safer than forcing every parsed item into a dispute. Utilization and report-item counts were zero because this extraction did not yield sufficiently reliable structured fields for those deterministic classifications.

Only confirmed and letter-eligible report-review categories can enable draft preparation. Utilization and general readiness actions cannot create letters.
