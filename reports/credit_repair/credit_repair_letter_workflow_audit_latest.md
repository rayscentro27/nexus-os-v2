# Credit Repair Letter Workflow Audit

- Current design preserved: `True`
- Old design restored: `False`
- No auto-send: `True`
- Specialist review required: `True`
- Client approval required: `True`

## 1. After Credit Report Upload

- File is uploaded through `DocumentUploadZone.tsx`.
- Metadata is inserted into `client_documents`.
- Status is `pending_review`.
- `goclear_review_status` is `pending_review`.
- `approval_required` is `true`.
- `client_visible` is `true`.

Current behavior: upload creates a document row only. It does not automatically create a credit repair case, report items, or letters.

## 2. Report Parsing / OCR

No real parser, OCR, or bureau connection exists in the current code path. The system does not read the report into tradelines automatically. The correct status is Pending GoClear Review or manual item entry.

## 3. Credit Report Items

Credit report items are currently created by:

- client/manual entry in `CreditRepairCaseEnginePanel`
- specialist/admin legacy workbench item creation in `CreditSpecialistWorkbench.jsx`

The helper function is `createManualReportItem(...)` in `src/lib/creditRepairCaseEngine.ts`.

## 4. Marking Items for Challenge

Client selects an item and clicks `I want this challenged`. This calls `markItemForChallenge(...)` and updates `credit_report_items.client_wants_challenged`.

## 5. Dispute Reason Selection

Client chooses a reason from `DISPUTE_REASON_LABELS`. The UI calls `selectDisputeReason(...)`, which creates a `credit_dispute_strategies` row and returns deterministic options.

## 6. Letter Options

Letter options are generated by:

- `generateDisputeLetterOptions(...)` in `src/lib/creditRepairCaseEngine.ts`
- `getDisputeOptions(...)` in `src/lib/disputeStrategyKnowledge.ts`

This is deterministic mapping, not AI generation.

## 7. Draft Creation Status

`createLetterDraftFromOption(...)` creates:

- `credit_dispute_letter_options` row with `status: specialist_review`
- linked legacy dispute item / letter draft through `creditRepairWorkflow`

The draft is not sent.

## 8. Specialist Review

Admin/specialist review happens in `/admin/credit-specialist`, powered by `CreditSpecialistWorkbench.jsx`.

Specialist can:

- review strategy
- review drafts
- approve for client
- move client-approved letters into DocuPost gate
- record/inspect outcomes

## 9. Client Approval

Client approval happens in `/client/dispute-review`.

The client can:

- view letter preview
- request edits
- approve letter

Approval calls the existing `clientApproveLetter(...)` workflow.

## 10. DocuPost / Send Request

DocuPost send request happens only after client approval.

Client route:

- `/client/dispute-review`
- `Authorize/send request`
- disabled unless letter status is approved

Existing function:

- `createDocuPostSendRequest(letter.id)` from `creditRepairWorkflow`

Safety:

- no auto-send
- approval gate preserved
- send request is not available before approval

## 11. Outcome Tracking

Outcome tracking exists through:

- `credit_dispute_outcomes`
- `recordDisputeOutcome(...)`
- outcome categories from `disputeStrategyKnowledge`
- admin specialist workbench outcome/strategy summary

Categories include deleted, corrected, verified, no response, needs escalation, client evidence needed, and not sent.

## 12. Missing for Full End-to-End Flow

- automatic credit report parsing/OCR
- automatic creation of report items from uploaded credit report
- direct linkage between uploaded evidence and specific report item/strategy
- richer specialist edit UI for case-engine letter options
- production partner/provider integrations
- safe document view URLs
- automated outcome ingestion from bureau/furnisher responses
