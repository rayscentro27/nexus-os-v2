# Plain-Language Quality Router Audit

**Date**: 2026-07-06
**Commit**: da7c6bb

---

## Current Router Flow

```
process_command(text)
  → slash command? → handler
  → followup intent? → cmd_followup()
  → alpha intent? → cmd_alpha_fallback()
  → DEFAULT → cmd_start() (command menu)
```

## Defects Found

### 1. GREETING_FALLS_TO_MENU
- **Location**: `process_command()` line 994-1066
- **Evidence**: "good morning" → no slash match → no followup match → no alpha match → `cmd_start()` returns full command menu
- **Root cause**: No greeting intent detection; default fallback is the command menu
- **Severity**: HIGH — first impression is a wall of commands

### 2. ALPHA_OVERROUTES_CASUAL_MESSAGES
- **Location**: `ALPHA_INTENT_PATTERNS` line 671-680, specifically `r"\balpha\b"` pattern
- **Evidence**: "alpha good morning" → matches `alpha_keyword` → routes to `cmd_alpha_fallback("good morning")` → creates research brief for a greeting
- **Root cause**: The word "alpha" alone triggers research routing; no distinction between "alpha as agent name" and "alpha as research request"
- **Severity**: HIGH — creates fake research from casual chat

### 3. HERMES_NO_DIRECT_ANSWER
- **Location**: `cmd_hermes()` line 311-339
- **Evidence**: `/hermes what is today's priority?` → only creates work order → returns "Hermes Request / Routed to: hermes_general / Work Order: wo_xxx"
- **Root cause**: Hermes handler is purely a work-order router; no direct advisory answer is generated
- **Severity**: HIGH — Ray asks a question, gets a routing receipt instead of an answer

### 4. STALE_REPORT_PRIORITY
- **Location**: `cmd_report()` line 116-159 reads from `reports/runtime/nexus_anytime_operator_report_latest.json`
- **Evidence**: Report still shows "Rotate Telegram token to restore full operator control" as priority #1 even though Telegram live polling is confirmed working
- **Root cause**: The JSON file is generated by another process and not updated to reflect current Telegram status
- **Severity**: MEDIUM — confusing stale info

### 5. RANKING_INCONSISTENCY
- **Location**: `cmd_alpha_fallback()` line 895 stores `last_alpha_recommendations` using unsorted `ideas` list; reply (line 908) shows `ranked[:3]` which is sorted by score
- **Evidence**: Ray sees top-3 sorted by score in the reply, but `which one should we do first?` uses `recs[0]` from the unsorted stored list
- **Root cause**: Two different orderings — display is sorted, storage is not
- **Severity**: MEDIUM — follow-up picks wrong recommendation

## Recommended Fix Plan

1. Add intent classification function with 9 categories (GREETING, CASUAL_AGENT_CHAT, ALPHA_RESEARCH_REQUEST, ALPHA_CONTEXT_FOLLOWUP, HERMES_ADVISORY, NEXUS_STATUS_OR_REPORT, APPROVAL_ACTION, WORK_ORDER_REQUEST, UNKNOWN_HELPFUL_FALLBACK)
2. Route greetings to natural responses (not menu, not Alpha brief)
3. Route casual Alpha chat to conversational replies (not research briefs)
4. Add direct advisory answer to Hermes before creating work order
5. Fix ranking: store sorted recommendations, not unsorted
6. Fix /report: check Telegram status dynamically, remove stale token priority
