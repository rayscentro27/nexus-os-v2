# Readiness Packet Approval and Delivery

Readiness packet content is generated as a structured draft from persisted journey, document, and readiness state. A draft is not client-visible.

Required stages:

1. draft
2. admin review
3. Ray Review when required
4. exact version approval
5. controlled portal delivery

Each packet includes source labels for Nexus-observed facts, client-provided facts, uploaded evidence, uncertainty, reviewer judgment, and recommendations. It carries disclaimers and is screened for guarantees, provider secrets, raw storage paths, persistent signed URLs, full account numbers, and invented facts.

Packet versions are unique per order. Reviewer attribution, approval timestamp, revision/rejection reason, and delivery timestamp are persisted. A delivered version is immutable and remains linked to the order and readiness context. No email, DocuPost submission, funding application, or automatic external action is created.

Tier 2 and Tier 3 consultation entitlement creates a scheduling request with pending admin confirmation when calendar configuration is absent. It is not represented as a confirmed live meeting.
